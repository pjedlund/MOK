# JEDEE Design System

## Overview

Also known as "Brand & Style".

JEDEE is the personal site of Johan Edlund — a UX/UI designer and front-end
developer based in Malmö, Sweden. The system is built for a slow, editorial
indieweb: long-form articles, a reading and listening log, and short notes
that link to one another with wikilinks. It is published with a public-domain
intent and signs off with **Soli Deo Gloria**.

The visual language pairs **slate-blue neutrals** with a single **earthy
orange** accent. It feels considered rather than flashy, closer to a printed
journal than a SaaS product. Type carries most of the personality: a stout
serif for display, a humanist sans for body, monospace reserved for code and
metadata. Depth is conveyed through tonal contrast and a one-pixel orange
border at the top of every page — never through shadows.

The system is fluid by construction. Every type and space step is generated
by [Utopia](https://utopia.fyi) and clamped between a 320 px and 1350 px
viewport, so the same tokens describe both phone and desktop without
breakpoint logic. Tokens are authored in `src/_data/designTokens/*.json`,
consumed by Tailwind to produce CSS custom properties, and pushed to
[Penpot](https://design.penpot.app) via `npm run penpot:tokens`. This
DESIGN.md is the third surface — for AI agents and humans who want the
system in one readable file.

## Colors

The palette is rooted in cool slate neutrals with a single warm-orange
driver. Accents (blue, green, red) stay desaturated so they sit beside the
orange without competing for attention.

- **Primary — Slate Ink (#495464):** The deep slate-blue from `base-darkest`.
  Used for body copy, headlines, and the default site theme color
  (`themeColor` in `meta.js`). Provides high contrast against the warm
  limestone background while staying softer than pure black.
- **Secondary — Source Orange (#d0621e):** The single accent driver.
  Reserved for link underlines, the input caret, the body's top border (in
  dark mode), the blockquote rule, and the conic-gradient logo. One screen
  rarely needs more than two or three uses.
- **Tertiary — Editorial Blue (#245375) and Moss Green (#626449):**
  Companion accents used in the conic logo gradient and reserved for
  category cues (e.g. listening, reading). Each has a `subdued` sibling for
  dark mode that drops chroma rather than lightness.
- **Neutral — Warm Limestone (#f4f4f2):** The default surface. A warm
  off-white that softens the slate ink and reads as paper rather than UI.

### Dark theme

The dark theme keeps the same accents and rotates the slate scale: the
background falls to `gray-900` (#141619), surfaces lift to `gray-700`, and
text rises to `gray-250` for readable contrast. The top border switches from
warm limestone (`base-light`) to a deep ember (`orange-900`), preserving the
single-orange-accent identity in inverted form. The dark-mode tokens use the
`-dark` suffix in the YAML front matter and mirror the light tokens
one-for-one.

### Design Tokens

See the `colors` block in the front matter. The semantic roles (`primary`,
`secondary`, `tertiary`, `neutral`, `surface`, `on-surface`, `headline`)
map onto the CSS custom properties in
`src/assets/css/global/base/variables.css`. The full grayscale and orange
scales (`gray-100` … `gray-900`, `orange-100` … `orange-900`) are exposed

## Typography

Type is the personality. Three Adobe Source families do all the work:

- **Display — Source Serif 4 (700):** Used for `h1`–`h3` and the wordmark.
  Stout, contemporary, with enough warmth to feel literary rather than
  corporate. Set tight (`leading-fine`, `leading-extrafine`) at large
  sizes.
- **Body — Source Sans 3 VF:** A humanist sans with a variable axis. Used
  for paragraphs, captions, navigation, and metadata. Set at
  `leading-standard` (1.4) with `letterSpacing: -0.04ch` — slightly tighter
  than default, to compensate for screen rendering.
- **Mono — Source Code Pro:** Reserved for code blocks, inline code, and
  rare technical metadata.

The size scale is fluid (Utopia) with nine steps from `step-min-2`
(13 → 16 px) to `step-6` (56 → 107 px). The same composite tokens drive
every breakpoint — no separate mobile/desktop type ramps.

### Design Tokens

The `typography` front-matter block lists the named composite styles —
`body`, `heading-1`–`heading-3`, `blockquote`, `citation`, `caption`,
`label-caps`, `code`. These mirror `src/_data/designTokens/typography.json`
and `src/assets/css/global/base/global-styles.css`. Sizes shown in the
front matter are the Utopia **max** values (the rendered size at the
1350 px viewport). At smaller viewports, the same token clamps down to its
min — see `src/_data/designTokens/textSizes.json` for the source ranges.

## Layout

The layout follows a **single-column, max-width** model: a 1360 px wrapper
(`--wrapper-width: 85rem`) with fluid gutters that shrink from 31 px down
to 14 px on phones. There is no rigid column grid — content flows in a
single editorial column, with cards and asides pulled out via CUBE CSS
compositions (Stack, Cluster, Switcher).

Spacing uses the Utopia fluid scale, with both fixed steps (`xs`, `s`,
`m`, `l`, `xl`, `2xl`, `3xl`) and **paired steps** (`s-m`, `m-l`,
`l-xl`, …). Paired steps interpolate across viewports more aggressively
than fixed ones — they're the right choice for vertical rhythm between
sections, where small phones want compactness and desktops want breathing
room. Fixed steps are used for component-internal padding where rhythm
should stay constant.

## Elevation & Depth

Depth is conveyed through **tonal layers and contrast**, never shadows. The
system is intentionally flat:

- The page background is **warm limestone** (`surface`), and elevated
  surfaces (cards, code blocks, top navigation in dark mode) lift to pure
  white (`surface-accent`) or `gray-700` in dark.
- The body has a **persistent 1 rem top border** in `border-top` — a
  warm-limestone band in light mode, an ember-orange band in dark. This
  is the single piece of "chrome" that survives across all pages.
- Interactive elements signal state through **color and underline weight**,
  not lift. Hover and focus never use box-shadow.
- The `:focus-visible` ring is a 3 px solid `currentColor` outline,
  offset by 0.3 ch (0.08 em on Firefox).

## Shapes

Corner radii are minimal. Most surfaces are square; a few utility shapes
soften.

- `rounded.sm` (0.1875 rem ≈ 3 px) — code blocks, inline code, small chips
- `rounded.md` (0.3 rem ≈ 5 px) — buttons, form inputs (when used)
- `rounded.full` (9999 px) — pill-shaped tags and the avatar
- The body, articles, headings, hr, and most layout primitives use **no
  corner radius** at all

The visual emphasis is on the orange one-pixel borders and the rhythm of
type, not on rounded containers.

## Components

### Body

The page itself. Sets the default font, color, and background; renders the
top accent border that frames every screen.

| Property | Light | Dark |
|---|---|---|
| `backgroundColor` | `colors.surface` | `colors.surface-dark` |
| `textColor` | `colors.on-surface` | `colors.on-surface-dark` |
| `borderTop` | 1 rem `colors.border-top` | 1 rem `colors.border-top-dark` |
| `accentColor` | `colors.accent-orange` | `colors.accent-orange` |

### Headings (h1–h3)

Set in Source Serif 4 at 700, with tight letter-spacing. `h1` uses
`heading-1` (largest, tightest line-height); `h2`/`h3` step down through
the size scale but keep `leading-fine` (1.2). Color is always `headline`,
which differs from body color in dark mode (lighter than text) for
hierarchy.

### Links

Inline anchors are styled at 0.2 ex underline thickness, offset 0.3 ch
from the baseline. Underlines are colored with `orange-600` in light mode
and `accent-red-subdued` on hover. Text color shifts to `accent-orange`
on hover. Underlines transition over 200 ms.

| State | textColor | underlineColor |
|---|---|---|
| Default | `colors.on-surface` | `colors.orange-600` |
| Hover | `colors.accent-orange` | `colors.accent-red-subdued` |
| Focus-visible | inherited | + 3 px outline at `currentColor` |

### Blockquote

A pulled, indented quote with a heavy left rule.

- Padding: `spacing.m-l` (fluid 14 → 31 px)
- Border-inline-start: 0.5 rem solid `colors.accent-orange`
- Type: `typography.blockquote` (44 px max), with the last child rendered
  as `typography.citation` (35 px max) for attribution
- Stack rhythm: paragraph children separated by `spacing.m-l`

### Inputs

Text inputs and textareas inherit body type. The caret is colored
`accent-orange`. Surface and corners follow `surface-accent` and
`rounded.md` when explicit container styling is applied.

### Horizontal rule

A 10%-width, 1 px bar in `surface-accent-2`. Acts as a quiet section
divider rather than a hard cut. Margin-block follows
`--flow-space, spacing.m-l`.

### Figure caption

Rendered in `typography.caption` (italic, 22 px max), centered, with a
small bottom padding so the caption sits under the figure rather than
the next paragraph. Color follows `on-surface-accent`.

### Selection

User text selection inverts the document: background becomes the body's
text color, foreground becomes the page background. This produces a
strong visual hit without introducing a new color.

### Focus ring

A 3 px solid `currentColor` outline, 0.3 ch offset. Reduced to 0.08 em on
Firefox to match its rendering. Never replaced by a shadow.

## Do's and Don'ts

- **Do** keep Source Orange the only saturated accent on a screen. One
  highlight is more legible than three.
- **Do** prefer paired spacing tokens (`s-m`, `m-l`) for vertical rhythm
  between sections; reserve fixed steps for inside-component padding.
- **Do** let type carry hierarchy — change size and weight before
  changing color.
- **Do** maintain WCAG AA contrast on body text (4.5:1) — the slate ink
  on warm limestone clears AA, and the dark theme's `gray-250` on
  `gray-900` clears AAA.
- **Don't** introduce shadows. The system is intentionally flat;
  elevation is tonal.
- **Don't** reach for accent-blue, accent-green, or accent-red as a
  call-to-action color. They are secondary cues for the conic logo and
  category accents only.
- **Don't** mix corner radii on the same surface. A code block uses
  `rounded.sm`; the page itself uses none. Don't add `rounded.md` to
  inline code.
- **Don't** introduce new font families. Source Serif 4, Source Sans 3,
  Source Code Pro cover display, body, and code respectively.
- **Don't** hard-code hex values in CSS. Reach for the CSS custom
  properties generated from the JSON tokens (or the matching Penpot
  variables) so a single source of truth flows everywhere.
